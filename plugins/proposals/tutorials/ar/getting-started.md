# البدء مع إضافة proposals

يأخذك هذا الدليل من مساحة عمل فارغة وينتهي بدورة عمل كاملة
اقتراح ← شريحة ← تنفيذ ← إغلاق، مع الحفاظ على انضباط قفل الملفات.
يفترض أن إضافة `proposals` مفعّلة (راجع `plugins/proposals/README.md`
للحصول على مقتطف JSON).

## 0. النموذج الذهني

**الاقتراح** هو ملف markdown مع ترويسة frontmatter. **الشريحة** هي
قسم مرقّم بداخله. تنسّق الإضافة بين كاتبَين لكل شريحة: أحدهما يطالب،
والآخر يحرّر. `auto_work` هو نقطة الدخول عالية المستوى: "ماذا أفعل
بعد ذلك؟".

```
docs/mcp-vertex/proposals/
├─ index.json          (يُعاد توليده بواسطة sync_proposals)
├─ p<N>-<عنوان>.md    (اقتراح)
│  ├─ ## Slices
│  │  ├─ s1-claim
│  │  ├─ s2-implement
│  │  └─ s3-close
```

## 1. البداية مع `auto_work`

يُعيد `auto_work` الشريحة القابلة للتنفيذ التالية عبر مخزن
الاقتراحات بأكمله، مع خطة مضغوطة ومرتبة. يجب تنفيذ الخطة
حرفيًا — دون ارتجال خطوات.

```json
// استدعاء أداة MCP
{ "tool": "proposals_auto_work", "args": {} }

// استجابة نموذجية (مختصرة)
{
  "state": "work",
  "proposalId": "l110",
  "sliceId": "s1-claim",
  "steps": [
    "افتح docs/mcp-vertex/proposals/l110-…md واختر الشريحة الذرية التالية.",
    "اطالب بملفاتها: proposals_agent_lock { action: \"claim\", … }.",
    "نفّذ هذه الشريحة فقط — لا شيء خارج الملفات المطالَب بها.",
    "تحقّق وفق بوابة المشروع (راجع get_validation_matrix إن وُجدت).",
    "ضع علامة على التقدم في الاقتراح، ثم proposals_sync_proposals.",
    "حرّر: proposals_agent_lock { action: \"release\", task_id }."
  ]
}
```

## 2. المطالبة بملفات الشريحة

تُسجّل أداة `proposals_agent_lock` من يملك أيّ مسارات لمدة الشريحة.
بدون مطالبة، سيرفض `sync_proposals` وسم الشريحة كمنجزة.

```json
{
  "tool": "proposals_agent_lock",
  "args": {
    "action": "claim",
    "files": [
      "apps/web/src/components/PluginPage.astro",
      "apps/web/src/data/capabilities.json"
    ]
  }
}
```

تحمل الاستجابة `task_id` يجب الاحتفاظ به حتى التحرير. عميلان
يطالبان بالملف نفسه ⇒ تعارض، لا تقدم. المزامرة مدعومة بنظام
الملفات (غير استشارية) وتبقى عبر إعادة تشغيل العملية.

## 3. تنفيذ الشريحة ثم التحقق

عدّل الملفات المطالَب بها فقط. شغّل بوابة المشروع:

```bash
bun run validate
```

إذا فشلت البوابة، أصلح الشريحة — لا توسّع المطالبة بصمت.

## 4. وسم التقدم والمزامنة

يقرأ `sync_proposals` ملفات الاقتراحات، ويتحقق من frontmatter + خطة
الشرائح، ويُعيد بناء `index.json`. إنه سريع وخامل (idempotent).

```json
{ "tool": "proposals_sync_proposals", "args": {} }
```

## 5. إغلاق الشريحة

```json
{
  "tool": "proposals_close_slice",
  "args": {
    "proposalId": "l110",
    "sliceId": "s1-claim"
  }
}
```

يُعيد هذا كتابة حالة الشريحة إلى `done` في الاقتراح، ويزيل
القفل ويُعيد مزامنة الفهرس. استدعِ `auto_work` مرة أخرى — سيُعيد
الشريحة التالية (أو `state: "idle"` إذا استُنفد المخزن).

## الأخطاء الشائعة

- **تعديل ملفات خارج المطالبة**: سيرفض `sync_proposals` وسم الشريحة
  كمنجزة. استخدم شريحة ثانية بمطالبتها الخاصة، أو قسّم الاقتراح.
- **تخطّي `sync_proposals`**: يصبح الفهرس قديمًا. يطلب العميل التالي
  "الشريحة التالية" ويحصل على الخاطئ.
- **نسيان التحرير**: يحجب قفل قديم المنسّق التالي لمدة `staleMs`
  (الافتراضي 30 ثانية). استدعِ
  `proposals_agent_lock { action: "gc" }` للتنظيف.
